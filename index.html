Selection Sort scans the entire unsorted portion of the array each pass to find the minimum element. This requires a nested loop that always performs roughly n² comparisons, giving O(n²) time.

Insertion Sort builds a sorted portion of the array by shifting elements as needed. In the worst and average cases, each new element might have to shift through the whole sorted side, creating n² total operations, so the time is O(n²).

#Operation

Merge Sort divides the array in half repeatedly, then merges the halves while comparing elements. The divide phase takes log n, and each level processes all n elements during the merge, giving O(n log n) time.
